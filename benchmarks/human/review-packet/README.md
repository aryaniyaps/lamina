# Blind human evaluation packet

Generated by `npm run bench:human-packet`.

## Instructions for raters

Methodology: **[Design A — ecological adoption](../../METHODOLOGY.md)**. Control implementations are post-Plan+implement; treatment implementations are post-Lamina-loop (verify/fix included in process, not in your rubric scope — score the code as given).

1. You will review pairs of **implementations** (labeled **A** and **B**) for each task — bundled application source, not design briefs
2. You do **not** know which arm (with/without Lamina) produced which implementation
3. Score each implementation independently using [`../../judges/rubric.md`](../../judges/rubric.md) — product-behavior in code, not UX research docs
4. Submit scores via Google Form ([setup guide](../google-form/SETUP.md)) or CSV template in this directory
5. Maintainers import with `npm run bench:import-google-form` or `npm run bench:import-human`

## Rubric criteria (1–5 each)

1. Domain / system structure
2. Invariants and product rules
3. Actors and permissions
4. Workflow quality
5. Scenario / edge coverage
6. Systems judgment
7. UX expression under rules
8. Brownfield fit
9. Implementation readiness
10. Overall product-behavior quality

## Files

- `packet.html` — Browser-friendly review UI
- `scores-template.csv` — Blank scoring sheet (CSV path)
- `human-eval-manifest.json` — Publish before raters start (verifiable eval ID + artifact hashes)
- `answer-key.json` — Arm mapping (private until scoring completes)

## Important

Synthetic / example human scores are **not** written by default and must never be cited.
Only CSV imports (`synthetic: false` in `human-scores.json`) enter the composite score.
