# LaminaBench Human Review Rubric

Score each criterion **1–5** (1 = poor, 5 = excellent). Raters are blind to whether an artifact is Control or Treatment.

## Criteria

1. **Business understanding** — Correct grasp of goals, users, and constraints
2. **UX completeness** — Required flows and requirements addressed
3. **Flow quality** — Logical user journeys with clear decision points
4. **Integration with existing product** — Fits brownfield context (N/A for greenfield → score on internal consistency)
5. **Edge-case coverage** — Empty, failure, permission, and boundary states
6. **Error handling** — Recovery paths and user guidance on failures
7. **Accessibility** — Keyboard, screen reader, contrast, and inclusive design
8. **Consistency** — Terminology, patterns, and structure throughout
9. **Implementation readiness** — Actionable enough for a developer to build from
10. **Overall UX quality** — Holistic judgment

## Scoring guide

| Score | Meaning |
|-------|---------|
| 1 | Missing or fundamentally wrong |
| 2 | Superficial; major gaps |
| 3 | Adequate; notable omissions |
| 4 | Strong; minor gaps only |
| 5 | Excellent; production-ready reasoning |

## Instructions

- Read the task description before scoring each artifact
- Do not reward formatting alone — evaluate reasoning depth
- Penalize shallow checklist filling without substantive UX analysis
- Record brief notes for scores ≤ 2 or ≥ 4
