## 2026-07-16 — Preflight audit hardening

- Scope: runtime isolation, raw evidence retention, telemetry, semantic validation, product snapshots, independent build/test checks, safe resumption, protocol hashing, fixture hashing, blinding, rating schemas, aggregation, claim gates, freeze inputs, and documentation.
- Reason: the initial v2 implementation passed its tests but did not prove host isolation, reconstructability, matched semantic completeness, stage validity, adjudication completeness, missing-data safety, or frozen outcome selection.
- Validity effect: invalidates all earlier v2 development and publication cells across every arm, track, cohort, task, and repeat.
- Existing affected cells at audit start: none under `benchmarks/results/v2`.
- Publication consequence: no publication task has been opened or executed; publication remains unavailable until independent custodian packages and external executor attestation exist.

### Defects corrected in this protocol revision

- Agent, reviewer, and transfer sessions now run in Bubblewrap namespaces that expose only the assigned workspace, a credential-only runtime home, fixed toolchain binaries, and the network resolver.
- Exact prompts, stdout/stderr, normalized event streams, final text, sessions, timestamps, token fields, tool calls, exit states, signals, timeouts, CLI versions, isolation attestations, validations, and file hashes are preserved.
- Contract v2 and the neutral structured contract receive equivalent semantic graph/readiness checks; oracle questions, business context, personas, reviewer count, contract immutability, review-only source immutability, snapshots, and application validation are enforced.
- Attempts and unfavorable failures are append-only; resumability requires matching protocol and cell-input hashes plus revalidation of the preserved evidence.
- Fixture cache keys now include recursive layer bytes rather than only layer paths.
- Judge-visible artifact packages are randomized and sanitized while native contract structure is preserved; the secret arm key must remain outside the judge package.
- Human rating summaries require independent rater IDs and a third adjudicator when a dimension gap exceeds one point.
- Contract and product ratings use separate arm-neutral rubrics; both product stages are retained.
- Aggregation refuses mixed tracks, cohorts, or protocol hashes, treats the task as the independent unit, records missing-data failures, and applies only the preregistered efficiency outcome.
