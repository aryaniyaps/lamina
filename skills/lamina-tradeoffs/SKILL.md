# Product decisions and tradeoffs

Record choices under `decisions.assumptions[]` or `decisions.forks[]` and classify them as structural, safety/integrity, reversible default, policy fork, desirability hypothesis, or evidence-backed.

For each decision, record the statement, provenance, confidence, status, affected graph references, and whether it blocks implementation.

Block readiness only when implementation would silently commit to consequential ownership, trust, money, destructive behavior, privacy, or regulatory policy. Make low-risk reversible defaults explicit and proceed. Never present a persona preference as confirmed demand.
