# Finding merge rules

Merge by graph reference and distinct risk, not wording similarity alone.

Priority order:

1. User or repository evidence.
2. Structural, safety, integrity, authority, and lifecycle contradictions.
3. Missing failure or recovery behavior.
4. Reversible UX observations.
5. Persona desirability hypotheses.

Never let a simulated preference override user or repository evidence. When reviewers disagree on product policy, create or update a decision fork. When they identify the same risk, keep one scenario and cite all supporting findings rather than duplicating it.
