## Glossary

**Affordance** — Perceived and actual properties determining how an object could be used ([Discoverability](../lamina-discoverability/SKILL.md))

**Affinity diagramming** — Grouping observations into thematic clusters to find patterns ([Research Synthesis](../lamina-research-synthesis/SKILL.md), [Interview Documentation](../lamina-interview-documentation/SKILL.md))

**Applied research** — Research serving a specific real-world goal ([Research Scoping](../lamina-research-scoping/SKILL.md))

**Behavior-first design** — Method focusing on user goals, expectations, and aptitudes ([Product Behavior](../lamina-product-behavior/SKILL.md))

**Breadcrumb** — Secondary navigation showing hierarchy from homepage ([Navigation](../lamina-navigation/SKILL.md))

**Causal research** — Post-launch cause-and-effect analysis ([Feature Discovery](../lamina-feature-discovery/SKILL.md))

**Cognitive effort minimization** — Pages and labels should not require unnecessary thought ([Content Design](../lamina-content-design/SKILL.md))

**Competitive research** — Systematic analysis of alternatives and market landscape ([Competitive Analysis](../lamina-competitive-analysis/SKILL.md))

**Conceptual model** — User's mental story of how a product works ([Discoverability](../lamina-discoverability/SKILL.md))

**Concise copy principle** — Halve web copy, then halve again ([Content Design](../lamina-content-design/SKILL.md))

**Constraints** — Limits on possible actions ([Discoverability](../lamina-discoverability/SKILL.md))

**Contextual inquiry** — Research in users' natural environment ([Field Research](../lamina-field-research/SKILL.md))

**Deep-page navigation orientation audit** — Five navigation questions answerable in five seconds from any page ([Navigation](../lamina-navigation/SKILL.md))

**Design requirements** — Capabilities a product must provide ([Requirements Definition](../lamina-requirements-definition/SKILL.md))

**Descriptive research** — Understanding context of a defined problem ([Feature Discovery](../lamina-feature-discovery/SKILL.md))

**Discoverability** — Whether users can determine what actions are possible ([Discoverability](../lamina-discoverability/SKILL.md))

**Evaluative research** — Testing proposed solutions ([Feature Discovery](../lamina-feature-discovery/SKILL.md), [Usability Evaluation](../lamina-usability-evaluation/SKILL.md))

**Featuritis** — Adding features without behavioral coherence ([Product Behavior](../lamina-product-behavior/SKILL.md))

**Feedback** — Communicating action results back to the user ([Discoverability](../lamina-discoverability/SKILL.md))

**Generative research** — Exploratory research before problem is defined ([Feature Discovery](../lamina-feature-discovery/SKILL.md))

**Gulf of evaluation** — Gap between system state and user's ability to interpret it ([Discoverability](../lamina-discoverability/SKILL.md))

**Gulf of execution** — Gap between user's goal and allowed actions ([Discoverability](../lamina-discoverability/SKILL.md))

**Heuristic evaluation** — Expert review against usability principles ([Usability Evaluation](../lamina-usability-evaluation/SKILL.md))

**Human-centered design** — Design process centered on human needs throughout ([Design Process](../lamina-design-process/SKILL.md))

**Implementation model** — How engineers think the product works internally ([Product Behavior](../lamina-product-behavior/SKILL.md))

**Mental model** — How users think something works based on experience ([Product Behavior](../lamina-product-behavior/SKILL.md))

**Mindless choices** — Unambiguous options requiring no thought to select ([Content Design](../lamina-content-design/SKILL.md))

**Mistake** — Error where the goal or plan is wrong ([Error Handling](../lamina-error-handling/SKILL.md))

**Perpetual intermediate** — Users who know basics but are not experts ([Feature Prioritization](../lamina-feature-prioritization/SKILL.md))

**Persona** — Archetypal user synthesized from research ([User Modeling](../lamina-user-modeling/SKILL.md))

**Primary persona** — The user the design must satisfy completely ([User Modeling](../lamina-user-modeling/SKILL.md))

**Product posture** — Product's behavioral stance relative to user attention ([Platform Posture](../lamina-platform-posture/SKILL.md))

**Research question alignment** — Cross-functional session to align on research questions ([Problem Framing](../lamina-problem-framing/SKILL.md))

**Right-sized research** — Matching research scope to decisions at hand ([Research Scoping](../lamina-research-scoping/SKILL.md))

**Satisficing** — Choosing first satisfactory option rather than optimal ([Content Design](../lamina-content-design/SKILL.md))

**Scan-first layout** — Designing pages for rapid scanning, not careful reading ([Content Design](../lamina-content-design/SKILL.md))

**Scenario** — Narrative describing persona achieving a goal ([Requirements Definition](../lamina-requirements-definition/SKILL.md))

**Signifier** — Signal communicating where action should take place ([Discoverability](../lamina-discoverability/SKILL.md))

**Slip** — Error where goal is correct but execution fails ([Error Handling](../lamina-error-handling/SKILL.md))

**Two-cycle diverge-converge process** — Discover, define, develop, deliver ([Design Process](../lamina-design-process/SKILL.md))

**Usability testing** — Observing users attempting tasks with a design ([Usability Evaluation](../lamina-usability-evaluation/SKILL.md))
