# Commands

Lamina exposes four slash commands. Each maps to a phase in the design → implement → verify loop.

## Quick path

```text
/lamina-init <domain>
/lamina-design <feature or flow>
# your agent implements from implement.md
/lamina-verify <base_url>
# your agent fixes from fix.md
/lamina-verify <base_url>
```

## Command reference

| Command | Phase | Purpose |
|---------|-------|---------|
| [`/lamina`](/commands/lamina) | Router | Auto-route to design, verify, or direct Q&A |
| [`/lamina-init`](/commands/lamina-init) | Charter | Domain scope — required before design or verify |
| [`/lamina-design`](/commands/lamina-design) | Design | Contract → `ready_to_build` + `implement.md` |
| [`/lamina-verify`](/commands/lamina-verify) | Verify | Persona walks → `findings[]` + `fix.md` |

## Prerequisite gates

| Command | Requires |
|---------|----------|
| `/lamina-init` | None |
| `/lamina-design` | Valid `.lamina/business-context.md` from init |
| `/lamina-verify` | Valid `.lamina/business-context.md` from init |
| `/lamina` (design/verify routes) | Same gates as the dispatched command |

<Callout type="info">
**Advanced:** The init gate cannot be bypassed. If `business-context.md` is missing or invalid, design and verify return a blocked response and stop.
</Callout>

## Typical sequence

```text
/lamina-init Boutique hotel booking marketplace
/lamina-design Guest checkout with payment hold and cancellation
# … your agent implements from implement.md …
/lamina-verify http://localhost:3000
# … your agent fixes from fix.md …
/lamina-verify http://localhost:3000
```

## Natural language

You can invoke Lamina without slash commands:

> "Use Lamina to design the onboarding flow for a team collaboration app."

The `/lamina` router interprets intent and dispatches to the right workflow.

## What Lamina does not do

| Action | Who handles it |
|--------|----------------|
| Write app source | Your coding agent |
| Implement fixes from `fix.md` | Your coding agent (normal coding session) |
| Pick your framework or database | You |
| Style UI (colors, typography) | UI skills or your agent |
