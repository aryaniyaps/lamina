# Quickstart

This walkthrough uses the HavenStay hotel booking domain from the Lamina demo. The same loop applies to any product on any stack.

## Quick path

1. Install Lamina in your agent
2. `/lamina-init` — domain charter
3. `/lamina-design` — contract at `ready_to_build`
4. Implement from `implement.md`
5. `/lamina-verify` with your dev server running
6. Fix from `fix.md` and re-verify

## Prerequisites

- Lamina installed in your AI coding agent ([Installation](/getting-started/installation))
- A project directory open in your agent

## 1. Install

```bash
npx skills install aryaniyaps/lamina
```

Run `/lamina` in your agent to confirm the skill loaded.

## 2. Domain charter

Scope the product domain before design. This step is required — design and verify are blocked without it.

```
/lamina-init Boutique hotel booking marketplace in the United States
```

**Expected output:**

```text
.lamina/
├── business-context.md
└── personas.yaml
```

Example personas from the HavenStay demo include `leisure_traveler`, `hotel_operator`, and `platform_admin`.

## 3. Design contract

Turn the domain into a build-ready contract:

```
/lamina-design Guest search, booking checkout with payment hold,
hotel onboarding, and reservation management
```

**Expected output:**

```text
.lamina/runs/<run_id>/
├── run.yaml          # status: ready_to_build
├── implement.md      # build handoff
└── report.md         # design narrative
```

Open `implement.md` — this is what your coding agent implements. Example opening:

```markdown
# HavenStay — Implement Brief

**Status:** `ready_to_build`
**Contract:** `.lamina/runs/havenstay-platform-2026-07-10/run.yaml`

Build a production-ready US hotel booking marketplace.
Use any stack, any UI library, any database.
```

## 4. Implement

Hand `implement.md` to your coding agent. **Lamina never writes app source** — your agent builds from the contract.

```
Implement the HavenStay booking platform from .lamina/runs/<run_id>/implement.md
```

Use any framework, database, and UI library. Lamina is stack-agnostic.

## 5. Verify

Start your dev server, then run verify against the live app:

```
/lamina-verify http://localhost:3000
```

<Callout type="info">
**Advanced:** Verify requires a running product at `base_url`. Lamina dispatches parallel subagents — one per primary actor — to walk the live app, check invariants, and probe reachability.
</Callout>

**Expected output when gaps exist:**

```text
.lamina/runs/<run_id>/
├── fix.md            # product fix brief
├── walkthrough/      # screenshots and evidence
└── run.yaml          # findings[] populated, status: complete
```

## 6. Fix and re-verify

Hand `fix.md` to your coding agent:

```
Fix the product issues listed in .lamina/runs/<run_id>/fix.md
```

Then re-run:

```
/lamina-verify http://localhost:3000
```

Loop until `findings[]` is empty and the contract holds.

## What you should have

After one full cycle:

| Artifact | Purpose |
|----------|---------|
| `.lamina/business-context.md` | Domain scope and constraints |
| `.lamina/personas.yaml` | Actor cast with permissions |
| `.lamina/runs/<id>/run.yaml` | Machine contract |
| `.lamina/runs/<id>/implement.md` | What your agent built |
| `.lamina/runs/<id>/fix.md` | Gaps found (if any) |
