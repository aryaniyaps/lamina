# Glossary

**Affordance** — Perceived and actual properties determining how an object could be used ([Discoverability](capabilities/discoverability.md))

**Affinity diagramming** — Grouping observations into thematic clusters to find patterns ([Research Synthesis](capabilities/research-synthesis.md), [Interview Documentation](capabilities/interview-documentation.md))

**Applied research** — Research serving a specific real-world goal ([Research Scoping](capabilities/research-scoping.md))

**Behavior-first design** — Method focusing on user goals, expectations, and aptitudes ([Product Behavior](capabilities/product-behavior.md))

**Breadcrumb** — Secondary navigation showing hierarchy from homepage ([Navigation](capabilities/navigation.md))

**Causal research** — Post-launch cause-and-effect analysis ([Feature Discovery](capabilities/feature-discovery.md))

**Cognitive effort minimization** — Pages and labels should not require unnecessary thought ([Content Design](capabilities/content-design.md))

**Competitive research** — Systematic analysis of alternatives and market landscape ([Competitive Analysis](capabilities/competitive-analysis.md))

**Conceptual model** — User's mental story of how a product works ([Discoverability](capabilities/discoverability.md))

**Concise copy principle** — Halve web copy, then halve again ([Content Design](capabilities/content-design.md))

**Constraints** — Limits on possible actions ([Discoverability](capabilities/discoverability.md))

**Contextual inquiry** — Research in users' natural environment ([Field Research](capabilities/field-research.md))

**Deep-page navigation orientation audit** — Five navigation questions answerable in five seconds from any page ([Navigation](capabilities/navigation.md), [Navigation](capabilities/navigation.md))

**Design requirements** — Capabilities a product must provide ([Requirements Definition](capabilities/requirements-definition.md))

**Descriptive research** — Understanding context of a defined problem ([Feature Discovery](capabilities/feature-discovery.md))

**Discoverability** — Whether users can determine what actions are possible ([Discoverability](capabilities/discoverability.md))

**Evaluative research** — Testing proposed solutions ([Feature Discovery](capabilities/feature-discovery.md), [Usability Evaluation](capabilities/usability-evaluation.md))

**Featuritis** — Adding features without behavioral coherence ([Product Behavior](capabilities/product-behavior.md))

**Feedback** — Communicating action results back to the user ([Discoverability](capabilities/discoverability.md))

**Generative research** — Exploratory research before problem is defined ([Feature Discovery](capabilities/feature-discovery.md))

**Gulf of evaluation** — Gap between system state and user's ability to interpret it ([Discoverability](capabilities/discoverability.md))

**Gulf of execution** — Gap between user's goal and allowed actions ([Discoverability](capabilities/discoverability.md))

**Heuristic evaluation** — Expert review against usability principles ([Usability Evaluation](capabilities/usability-evaluation.md))

**Human-centered design** — Design process centered on human needs throughout ([Design Process](capabilities/design-process.md))

**Implementation model** — How engineers think the product works internally ([Product Behavior](capabilities/product-behavior.md))

**Mental model** — How users think something works based on experience ([Product Behavior](capabilities/product-behavior.md))

**Mindless choices** — Unambiguous options requiring no thought to select ([Content Design](capabilities/content-design.md))

**Perpetual intermediate** — Users who know basics but are not experts ([Feature Prioritization](capabilities/feature-prioritization.md))

**Persona** — Archetypal user synthesized from research ([User Modeling](capabilities/user-modeling.md))

**Primary persona** — The user the design must satisfy completely ([User Modeling](capabilities/user-modeling.md))

**Product posture** — Product's behavioral stance relative to user attention ([Platform Posture](capabilities/platform-posture.md))

**Research question alignment** — Cross-functional session to align on research questions ([Problem Framing](capabilities/problem-framing.md))

**Right-sized research** — Matching research scope to decisions at hand ([Research Scoping](capabilities/research-scoping.md))

**Satisficing** — Choosing first satisfactory option rather than optimal ([Content Design](capabilities/content-design.md))

**Scan-first layout** — Designing pages for rapid scanning, not careful reading ([Content Design](capabilities/content-design.md))

**Scenario** — Narrative describing persona achieving a goal ([Requirements Definition](capabilities/requirements-definition.md))

**Signifier** — Signal communicating where action should take place ([Discoverability](capabilities/discoverability.md))

**Slip** — Error where goal is correct but execution fails ([Error Handling](capabilities/error-handling.md))

**Mistake** — Error where the goal or plan is wrong ([Error Handling](capabilities/error-handling.md))

**Two-cycle diverge-converge process** — Discover, define, develop, deliver ([Design Process](capabilities/design-process.md))

**Usability testing** — Observing users attempting tasks with a design ([Usability Evaluation](capabilities/usability-evaluation.md), [Usability Evaluation](capabilities/usability-evaluation.md))
