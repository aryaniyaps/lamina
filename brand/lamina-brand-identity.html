Section 01

Strategic Foundation

Category & Position

Lamina is a skill for developers who build with AI coding agents — Cursor, Claude Code, Codex, Gemini, Pi. It helps you know what to build and iterate faster: edge cases, UX gaps, and product states handled upfront in a contract your agent implements, then verified visually after you ship. Not for designers. Not a UI generator. Never writes your app source.

ProblemWhat Lamina handles
Edge cases discovered lateScenarios and recovery paths in the design contract
Invalid product statesEntities, invariants, permissions, lifecycle rules
UX gaps in flowsNavigation, forms, empty/error states tied to domain
Built ≠ designedVisual walkthrough — subagents read flows from screenshots

Onliness Statement

Lamina is the ONLY open-source skill built for developers who code with AI — iterate faster on knowing what to build, with edge cases and product states handled upfront and verified visually after you ship. Plugs into Cursor, Claude Code, Codex, Gemini, Pi. Any stack, any UI library.

The UX Layer Principle

Grey is the brand. Highlighter is the only color. Dotted lines mean UX thinking happened here.

Visual hierarchy

LayerVisualJob
BackgroundDark grey scaleNeutral UX structure
ContentDotted wireframe cardsFlows, personas, specs
AccentHighlighter #FACC15Active path, CTAs, labels
Mascot3D meerkat (full color)Personality, warmth

Why grey + one accent (branding skill)

  • Airey: Color supports structure — grey form is primary
  • Hyndman: Grey = still/calm (wireframe); meerkat = sparkling (personality)
  • Sharp: Grey + dotted lines + highlighter = distinctive asset combo
  • Highlighter semantics: UX workshop markup — not luxury gold, not SaaS blue

What we signal

we are

Know what to build · iterate faster · edge cases upfront · visual verify · open source

we are not

UI styling · app source writes · pretty prototypes · AI magic

lamina 06 — Mascot
Section 06

Mascot — 3D Meerkat

Character: Cute, sentinel, competent. Watches flows, catches edge cases. Tan fur, gold/highlighter chest chevron. Lives in a grey wireframe world — the only full-color organic element.

Logo vs mascot

AssetFormatUse
Layer markSVG vectorFavicon, navbar, header lockup, GitHub avatar
3D meerkatRendered PNG/WebPLanding sections, empty states, 404, social

Rule: Mascot never appears in the logo, wordmark, or favicon. Personality and product identity stay separate.

Landing page poses (max 4)

Hero — Sentinel

Upright, watching wireframe demo. Right side of hero.

How it works — Pointing

Gesturing at a flow step or persona card.

Edge cases — Curious

Leaning in, examining a dotted edge-case card.

CTA — Welcoming

Small wave beside install block.

Pose rules

  • Same render style across all poses (lighting, material, camera)
  • Dark grey (#09090B) background in every render
  • Pose reacts to UX content — never decorative
  • Max one meerkat per viewport height
  • Never use mascot in logo lockup, favicon, or navbar — layer mark only
